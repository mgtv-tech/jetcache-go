# 版本与功能可用性

本文档用于说明关键用户能力从哪个版本开始可用。

## 功能时间线

| 能力 | 起始版本 | 说明 |
| --- | --- | --- |
| 泛型 `MGet` 支持回源函数（`fn`）并使用远程 pipeline 优化 | `v1.1.0+` | 在分布式缓存场景下，远程批量读取通过 pipeline 降低网络往返开销。 |
| 更新后跨进程本地缓存失效（全 Go 进程） | `v1.1.1+` | 通过 `WithSyncLocal(true)` + `WithSourceId(...)` + `WithEventHandler(...)` 传播失效事件。 |

## 兼容性建议

- 若服务版本 `< v1.1.0`，不要依赖泛型 `MGet` 的回源函数行为。
- 若服务版本 `< v1.1.1`，本地缓存跨进程失效需由业务侧自行实现。
- 生产升级建议在 `go.mod` 固定版本标签，并重点回归 `Once`、`MGet`、自动刷新、本地同步链路。

## 文档范围说明

仓库主文档默认描述最新稳定版本行为。
若你的线上版本低于文档中标注的功能起始版本，请以当前运行版本行为为准。
