# 术语说明

本页统一 `jetcache-go` 文档中的核心术语。

| 术语 | 定义 |
| --- | --- |
| 缓存穿透 | 不存在的数据被反复请求，缓存无法命中，请求持续打到后端。 |
| 缓存击穿（热点 key 击穿） | 热点 key 过期后并发回源，后端瞬时压力激增。 |
| 缓存雪崩 | 大量 key 在短时间集中失效或不可用，导致后端过载。 |
| Singleflight | 同一 key 同一时刻只允许一个回源执行，其它并发共享结果。 |
| 自动刷新 | 对选定 key 进行后台刷新，降低过期瞬时冲击。 |
| 降级 | 在缓存或后端异常时，尽量返回部分可用能力。 |
| 两级缓存 | 本地缓存 + 远程缓存的分层读写路径。 |
| 占位符（not-found 占位符） | 用于表示“未找到”的哨兵值，用来防穿透。 |
